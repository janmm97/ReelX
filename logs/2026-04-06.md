Veo image-to-video models use the `/api/v1/veo/generate` endpoint with `FIRST_AND_LAST_FRAMES_2_VIDEO` generation type, separate from the generic `/jobs/createTask` path used by all other models.

### "Use this image to generate a video" button

Added a shortcut button beneath every generated image. Clicking it:
1. Fetches the generated image and wraps it as a `File` object
2. Pre-fills the upload preview in the Image-to-Video panel
3. Copies the image prompt into the prompt box
4. Switches the tab to **Video → Image to Video** mode

### API routing refactor (`generate-video/route.ts`)

Introduced a third dispatch path alongside Runway and Veo:
- `RUNWAY_MODELS` → `/api/v1/runway/generate` + Runway polling
- `VEO_MODELS` → `/api/v1/veo/generate` + Veo polling
- `GENERIC_MODELS` → `/api/v1/jobs/createTask` + generic `recordInfo` polling
